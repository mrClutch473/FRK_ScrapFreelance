# 📌 О проекте

**FRK ScrapFreelance** — полностью автоматизированная система мониторинга фриланс-бирж с интеграцией искусственного интеллекта. Проект решает задачу поиска и первичной обработки заказов без участия человека: парсер собирает заказы с биржи, нейросеть анализирует их релевантность и генерирует готовые отклики, а результаты приходят прямо в Telegram.

Система работает непрерывно на VPS и управляется через Telegram-бота.

---

# ⚙️ Как работает система

## 1. Парсинг заказов

Каждые 30 минут планировщик запускает парсер, который автоматически обходит все страницы раздела на **Kwork.ru**. Playwright открывает браузер в фоновом режиме (`headless`), имитирует реальный браузер через `User-Agent` и дожидается полной загрузки динамического контента. BeautifulSoup извлекает из HTML структурированные данные по каждому заказу.

По каждому заказу собирается:

* название и ссылка
* полное описание (включая скрытые блоки)
* желаемый и максимальный бюджет
* количество откликов и статус активности
* статистика заказчика: количество проектов на бирже и процент найма

---

## 2. AI-анализ и генерация откликов

Через 5 минут после парсинга запускается анализатор. Каждый новый заказ из базы данных прогоняется через **OpenRouter API** (доступ к моделям GPT/Claude/Mistral и др.) по двум отдельным промптам:

* `FRK_ANALYZ_PROMT.txt` — оценивает релевантность заказа для исполнителя
* `FRK_RESPONSE_PROMT.txt` — генерирует персонализированный отклик под конкретный заказ

---

## 3. Telegram-бот

Управление и получение результатов реализовано через Telegram-бота на **Aiogram**. Бот уведомляет о новых подходящих заказах и отправляет сгенерированные отклики прямо в чат.

---

## 4. Планировщик задач

Все процессы координирует **APScheduler** с асинхронным бэкендом:

| Задача | Расписание |
|---|---|
| Парсинг заказов | каждые 30 минут |
| AI-анализ | каждые 30 минут (с задержкой 5 мин от парсера) |
| Очистка старых заказов | каждый день в 03:00 |

---

## 5. База данных

Все заказы сохраняются в локальную SQLite-базу. Перед сохранением проверяется уникальность заказа по `id` — дублей не возникает. Ежедневная задача очищает устаревшие записи.
